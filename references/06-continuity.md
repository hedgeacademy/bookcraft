# Phase 6：全书连续性维护

长篇书稿必须保持连续统一。持续维护以下一致性：

- 主题一致
- 术语一致
- 结构一致
- 案例使用一致
- 论证节奏一致
- 写作风格一致
- 读者对象一致
- 概念定义一致
- 章节之间的承接关系一致

如果前文提出某个概念，后文应沿用同一名称。除非用户明确要求更换术语，否则不随意改名。

如果前文承诺后文展开，后文必须兑现。

如果后文依赖前文概念，应明确建立承接关系，避免读者产生断裂感。

## CONTINUITY.md

及时更新 `CONTINUITY.md`：

```markdown
# 全书连续性记录

## 核心概念
- [概念]：[定义和首次出现章节]

## 术语规范
- 使用：[统一术语]
- 避免：[不再使用的旧称]

## 已完成章节
- 第 X 章：[完成状态和主要内容]

## 未完成问题
- [待展开问题]

## 待补充案例
- [章节]：[需要的案例]

## 待核查引用
- [章节]：[待核查内容]

## 后续写作提醒
- [提醒事项]
```

## 更新时机

以下情况必须更新连续性文件：

- 新增章节
- 修改全书方向
- 改变核心术语
- 新增重要概念
- 调整章节顺序
- 引入跨章节案例
- 发现前后文承接问题
