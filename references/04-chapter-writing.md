# Phase 4：逐章写作

逐章推进书稿，不要一次性生成缺乏控制的整本书。

每章写作前，先确认本章功能：

- 本章承接的前文问题
- 本章核心论点
- 本章关键案例
- 本章需要解释的概念
- 本章需要回应的读者疑问
- 本章结尾要引出的下一章内容
- Phase 1 中规划的节数、小节数、预计字数和引文安排

每章正文应包括：

- 章标题后的引文，不少于 700 字，默认目标 1300 字；正文中直接呈现引文内容，不得写“章前引文”四个字或添加同名标题
- 有吸引力的开头钩子
- 清晰的问题意识
- 稳定的论证主线
- 真实案例或资料支撑
- 必要的概念解释
- 段落之间自然过渡
- 推动全书继续前进的结尾

写作时避免空泛概念、机械罗列、案例堆砌和风格漂移。每一章都必须服务于全书主线，而不是变成孤立文章。

## 按大纲规划写作

写作阶段必须遵循 Phase 1 的大纲规划：

- 正文直接从第 1 章开始。
- 每章按规划节数展开；如需减少或增加节数，先说明原因并同步更新 `OUTLINE.md`。
- 每节按规划小节数展开；内容较少而不分小节时，必须在章节说明中说明原因。
- 每章字数不少于 10000 字，默认目标 13000 字。
- 每节字数不少于 3500 字，默认目标 4500 字。
- 每小节字数不少于 1300 字，默认目标 2000 字。
- 每章必须包含章标题后的引文，不少于 700 字，默认目标 1300 字，但不得显示“章前引文”标签。
- 每节必须包含节前引文，不少于 350 字，默认目标 700 字。
- 小节默认不要求引文；如使用小节引文，应服务于论证，不要为形式而添加。
- 字数和引文长度可根据用户目标、章节功能和材料密度合理调整，但调整后要更新 `OUTLINE.md` 和 `CONTINUITY.md`。
- 章末不得写“第 X 章完”“本章约 X 字”“本章字数”等写作进度或统计提示；这些信息只能记录在项目管理文件中，不得进入正文或 Word 成品。
- 节编号必须从所属章号开始，例如第 1 章下依次使用 `1.1`、`1.2`，不得因序言、自序、前言或目录而变成 `2.1`、`3.1`。

## 图书内容规范参数

写作时读取 `book.toml` 中的 `[bookcraft.content_rules.*]` 和 `[bookcraft.normalization]`，按参数执行以下规则。本模块负责落实“内容规范”的写作部分。

### 书名、目录、内容简介和前言

- 书名应根据读者对象调整：入门级读者要求通俗易懂、适当活泼；专业级读者要求专业性强、关键词明显。
- 书名必须突出本书卖点。
- 目录通常到三级标题，即第 X 章、X.X、X.X.X。
- 各级标题应言简意赅、紧扣书名，体现全书内容框架和重点。
- 内容简介目标约 200 字，概括主要内容、特点、定位和读者对象。
- 前言一般不超过 2 页，说明技术发展概况、写作初衷、写作必要性、参与人员和致谢人员。

### 语言风格

- 优先使用主谓宾结构，避免倒装或名词堆叠造成理解负担。
- 使用书面化语言，避免口语化和网络化表达。
- 除非确有必要，不使用疑问句。
- 除非实例需要，正文避免使用“我、你、他、我们、你们、他们”等人称代词；需要作者说明时使用“笔者”。
- 尽可能用“其”替代“它/它们”，用“及其”替代“它的/它们的”。
- 标点明确，语句不得有逻辑歧义。

### 标题与段落承接

- 第 1 级标题与第 2 级标题之间、第 2 级标题与第 3 级标题之间，必须有引出性的介绍文字。
- 不允许连续出现两个标题。
- 除非确有需要，不出现第 4 级标题。
- 第 2 级节题中不要重复第 1 级节题内容。

### 界面、操作和术语

- 对话框、文本框、下拉列表框、按钮、菜单和选项名称必须写全，并使用中文双引号。
- 每个操作步骤只写一项操作，并写清该步骤的操作结果。
- 可以简化重复步骤，但不能省略必要步骤。
- 技术术语翻译必须从全书开头到结尾保持一致。
- 英文界面软件涉及界面时保留英文名称；英文词首次出现时可在括号中标注中文。

### 图、表、代码和物理量

- 图必须紧跟正文第一次提及之处，正文中必须出现“如图 X-X 所示”“见图 X-X”等指引。
- 图题术语必须与正文一致，图题必须准确描述图内容。
- 图上的说明文字应在 Word 中输入，不要做成图片。
- 代码只列出说明正文所需的部分，避免用大段无用代码拼凑篇幅。
- 代码应有良好缩进和必要注释，并确保可运行。
- 物理量必须使用阿拉伯数字并加法定计量单位；数量和单位之间留空格。

### 规范词

按 `[bookcraft.normalization]` 执行词语统一。例如：

- 座标 → 坐标
- 缺省值 → 默认值
- 帐号 → 账号
- 图象 → 图像
- 存贮 → 存储
- 其它 → 其他
- 仅使用 `E-mail`、`电子邮件`、`Windows` 等规范拼写
